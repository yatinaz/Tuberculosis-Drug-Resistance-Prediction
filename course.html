The Patient Records
— DNA as Binary Data

Imagine a blood bank with 3,393 vials. Each vial has 222 test results — each either positive (1) or negative (0). A SNP mutation is present or absent. That's it. Binary. Clean. Some vials don't have a resistance label for a particular drug — they're marked -1 ("not tested"). Before training, we throw those unlabeled vials out.

What the Data Looks Like

TB-003100…-1 (not tested)

Each column is a SNP or indel — a specific mutation at a specific position in the bacterium's genome. 222 columns × 3,393 rows = 752,646 individual data points.

Training Day
— 220 Models in One Script

Medical licensing boards don't test new doctors on their own patients. They use standardized cases the doctor has never seen. K-fold cross-validation works the same way — split the data into 5 groups, train on 4, test on the 1 held out, rotate, repeat. Each model gets tested 5 times on data it's never seen.

The 4 Models

🌲
The Gradient Surfer
XGBoost
Builds trees one at a time, each one fixing where the previous went wrong. Like a sculptor who keeps refining the same block of marble.
🌳
The Democracy
Random Forest
Builds 500 independent trees and lets them vote. Chaotic individually, wise collectively.
📈
The Minimalist
Logistic Regression
Draws a single straight line through the data. Shockingly competitive for how simple it is.
🎯
The Arbitrator
Ensemble
Averages the probability estimates from all three. Hedges bets, rarely the worst performer.

Module 5 of 6

Metrics That
Actually Matter

If 90% of your email is ham and 10% is spam, a model that labels everything "ham" gets 90% accuracy. It's useless — it catches zero spam. The same trap exists here: if only 12% of patients are susceptible to OFLX, a model that labels everyone "resistant" gets 88% accuracy. Impressive number. Dangerous outcome.

What AUROC Actually Means

The AUROC curve shows the tradeoff between catching resistant cases (True Positive Rate) and wrongly flagging susceptible ones (False Positive Rate). The area under this curve = AUROC. 1.0 is perfect. 0.5 is a coin flip. 0.99 means the model ranked every resistant patient above every susceptible one.

False Positive Rate → ↑ True Positive Rate

The Clinical Threshold Trade-off

At what probability do we call someone "resistant"? The threshold is a clinical decision:

❌ Miss a resistant case (False Negative)

Patient gets a useless drug for 2–6 weeks. Disease progresses. MDR-TB may worsen. Lives at risk.

⚠️ Wrongly flag as resistant (False Positive)

Patient gets an alternative drug — suboptimal but not fatal. We'd rather be cautious.

Conclusion: We'd rather have false positives than false negatives. Set threshold so sensitivity ≥ 90%.

Code ↔ English — Threshold Finding

tb_pipeline.py — threshold finding (lines 174–184) 
fpr, tpr, thresholds = roc_curve(y_true, y_pred_proba)
idx = np.argmin(np.abs(tpr - target_sensitivity))
threshold = thresholds[idx]
y_pred = (y_pred_proba >= threshold).astype(int)
tn, fp, fn, tp = confusion_matrix(y_true, y_pred).ravel()
specificity = tn / (tn + fp) if (tn + fp) > 0 else 0
→Calculate the full sensitivity–specificity curve at every possible threshold
→Find the point on the curve closest to 90% sensitivity
→Lock in that threshold value (e.g., predict 'resistant' if probability > 0.35)
→Apply the threshold: convert probability scores into 0/1 predictions
→Unpack the confusion matrix into four buckets: TN, FP, FN, TP
→Specificity = what fraction of actually-susceptible patients did we correctly identify
